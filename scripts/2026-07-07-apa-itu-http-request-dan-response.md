## Hook (0–4 detik)

**Subtitle:** HTTP request dan response itu dasar web. Kalau ini belum jelas, backend dan frontend bakal susah dipahami.

## Body (3–46 detik)

### Poin 1
**Judul:** Request = pesan dari browser ke server
**Isi:** Setiap kali kamu buka halaman atau klik tombol, browser kirim HTTP request. Isinya ada method (GET, POST), URL, dan header.

### Poin 2
**Judul:** Server proses request lalu kirim response
**Isi:** Server baca request, jalankan logic (query database, validasi, dll), lalu balikin HTTP response. Body response bisa HTML, JSON, atau file.

### Poin 3
**Judul:** Status code kasih tahu hasilnya
**Isi:** Angka di response memberi petunjuk. 200 berarti sukses, 404 tidak ketemu, 500 error di server. Cek status code dulu saat debug API.
