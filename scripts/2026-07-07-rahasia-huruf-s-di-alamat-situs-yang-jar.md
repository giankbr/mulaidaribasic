## Body (3–46 detik)

### Poin 1
**Judul:** S di HTTPS artinya Secure
**Isi:** HTTP biasa mengirim data apa adanya tanpa enkripsi. Dengan HTTPS, semua teks yang kamu kirim diubah jadi kode acak. Orang lain tidak bisa membaca isi pesan kamu.

### Poin 2
**Judul:** Kunci rahasia yang mengunci data kamu
**Isi:** Server dan browser kamu saling tukar kunci digital saat pertama kali terhubung. Data seperti password dikunci rapat di asal, baru dibuka saat sampai di tujuan.

### Poin 3
**Judul:** Wajib ada saat bikin website sendiri
**Isi:** Browser modern sering menandai website tanpa HTTPS sebagai tidak aman. Kalau kamu bikin web, wajib pasang SSL supaya pengunjung merasa aman dan data mereka terlindungi dari pembajakan.

## Caption

Pernahkah kamu memperhatikan ada huruf S di belakang alamat HTTP? Kadang kita cuma tahu kalau itu lambang gembok di browser, tapi jarang tahu apa fungsinya.

Sebenarnya, huruf S itu singkatan dari Secure. Zaman dulu, website pakai HTTP biasa untuk mengirim data. Masalahnya, data itu dikirim dalam bentuk teks polos. Jadi, kalau ada orang jahat yang menyadap koneksi internet kamu, dia bisa langsung membaca password atau data pribadi yang kamu kirim.

Nah, HTTPS hadir untuk mengatasi masalah itu. Cara kerjanya pakai sistem enkripsi. Jadi sebelum browser kamu mengirim data ke server, datanya akan diacak dulu jadi kode yang nggak bisa dibaca. Server yang dituju punya kunci rahasia untuk membuka kode tersebut.

Itulah kenapa kalau kamu sedang bikin website sendiri, kamu wajib memasang sertifikat SSL. Tanpa sertifikat ini, browser seperti Chrome akan memberi peringatan merah bahwa website kamu tidak aman.
